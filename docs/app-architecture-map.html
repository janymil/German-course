🚨 Priority Gap Analysis — What to Fix

1. Story Unlock Gate HIGH

Stories are completely open — no progression. Lesson 1–3 completion should unlock Story 01, Lesson 4–6 unlocks Story 02, etc. Currently a beginner can read the hardest story first with no guidance.

2. StudyCoach is Blind HIGH

The StudyCoach panel reads ZERO data from Cerebro. It can't recommend anything. It should read completedLessons, vocabSeen (sorted by weakest), storiesRead, and dailyXP to generate a "Today's Plan" recommendation.

3. Grammar Guide is Dead-End HIGH

GrammarGuide is a static reference with no connection to the rest of the app. Lesson mistakes should highlight relevant sections. Grammar card clicks in StoryReader should be tracked in Cerebro.

4. VocabTrainer Missing Source Tags MED

User doesn't know WHY a word is in their deck. All words from lessons, stories, and chat look identical. readingVocab already has storyId — VocabTrainer should display it.

5. Wrong Answers Not Drilled MED

Exercises you get wrong are not boosted in SRS priority. A word you failed 3 times should surface first in VocabTrainer. Currently all words share equal priority.

6. AI Chat Not Level-Aware MED

The AI system prompt doesn't include the user's completed lessons or current level. It should adapt difficulty, vocabulary, and grammar to what the user has actually learned so far.

7. Passive Listening = 0 XP LOW

PassivePhase rewards nothing. A 20-minute listening session should count toward daily XP and streak. Otherwise users have no incentive to use it.

8. Dashboard Missing Story Data LOW

Dashboard shows lesson stats but ignores storiesRead and readingVocab. A "Recently Read" widget and "Words from Reading" count would complete the picture for the Study Coach view.
